# 작업 보고서 - DummyDataGenerator PoC

## 요청 받은 작업

"Dummy 데이터 생성 Tool" PoC. 테스트를 위한 Dummy Data를 생성하는 도구이며, 생성된 데이터는 연결된 DB에 추가되어야 함.

## 해석

- "연결된 DB에 추가"를 다른 PoC(DataPersistence/DataMonitor)와 동일한 JSON 스키마·경로 구조로 해석하여,
  이 도구 하나만으로 완결되지 않고 다른 PoC의 데이터 폴더에 직접 주입할 수 있도록 `--data-dir` 옵션을 제공함.
- 무작위 생성이라도 참조 무결성(주문이 존재하는 시료만 참조)은 반드시 지켜야 실제 시스템에서 쓸모있는 더미 데이터라고 판단함.

## 변경 요약

- `generator.py`: 시료/주문 더미 데이터 생성, 기존 데이터 대비 자동 채번, 참조 무결성 유지

## 핵심 코드 요약

- `next_id()`가 기존 레코드에서 숫자를 추출해 최대값+1로 다음 ID를 계산하므로, 여러 번 실행해도 ID 충돌이 없다.
- `generate_orders()`는 `existing`(기존 시료 목록)에서만 `sample_id`를 선택하여, 존재하지 않는 시료를 참조하는 주문이 생성되지 않는다.

## 요청사항 충족 여부 체크리스트

- [x] 더미 시료(Sample) 데이터 생성
- [x] 더미 주문(Order) 데이터 생성 및 기존 시료 참조
- [x] 생성된 데이터가 지정된 DB(JSON 폴더)에 추가됨
- [x] `--seed`로 재현 가능한 생성 확인, DataMonitor로 결과 조회하여 스키마 호환성 검증 완료
